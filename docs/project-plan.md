# План проекта: Telegram-бот для психоневрологической диагностики

## Этапы разработки

### Этап 1: Настройка окружения (1 день)
- Создание виртуального окружения (Python 3.11)
- Установка зависимостей (aiogram, httpx, pydantic, tiktoken, python-dotenv)
- Создание файла .env с переменными (TELEGRAM_BOT_TOKEN, OPENROUTER_API_KEY)
- Создание структуры папок

### Этап 2: Базовая структура и FSM (2 дня)
- Реализация основного бота (main.py, handlers/)
- Настройка команд (/start, /help)
- Реализация конечного автомата (states.py) с этапами:
  - symptom_collection
  - context_collection
  - evaluation
  - recommendations
- Хранение данных пользователя в JSON (storage.py)

### Этап 3: Интеграция с OpenRouter (2 дня)
- Создание клиента OpenRouter (openrouter_client.py)
- Реализация роутера моделей (model_router.py)
- Обработка ответов, retry, таймауты

### Этап 4: Структурированный вывод (1 день)
- Определение Pydantic моделей (models.py)
- Парсер JSON-ответов (parser.py)
- Валидация и преобразование в объекты

### Этап 5: Управление контекстом (2 дня)
- Загрузка истории из JSON
- Подсчет токенов (tiktoken)
- Формирование промпта (context_manager.py)

### Этап 6: Безопасность и логирование (1 день)
- Фильтрация экстренных фраз (safety.py)
- Добавление дисклеймера
- Логирование инцидентов в incidents.json

### Этап 7: Тестирование (2 дня)
- Написание unit-тестов для модулей
- Интеграционное тестирование с мок-сервером
- Ручное тестирование через бота

### Этап 8: Документация и развертывание (1 день)
- Написание README.md
- Подготовка Dockerfile (опционально)
- Инструкция по запуску

## Приоритеты
1. Работоспособное MVP с фиксированными вопросами.
2. Гибкая смена моделей.
3. Безопасность и дисклеймеры.
4. Возможность масштабирования (потом перейти на SQLite/PostgreSQL).
