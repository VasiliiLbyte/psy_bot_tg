# Управление контекстом

## Цель
Сохранять контекст диалога для AI-агента, чтобы модель видела историю общения и собранные данные.

## Принципы
- История хранится в `data.json` в массиве `history` для каждого пользователя.
- В `history` сохраняются последние N сообщений (по умолчанию 20).
- При формировании промпта используется системный промпт + собранные данные + история.
- Подсчёт токенов через `tiktoken` (модель `cl100k_base`).
- Если лимит токенов превышен (например, 100K), обрезаются самые старые сообщения.

## Формат промпта
System: {system_prompt}

Собранные данные:
{collected_data}

История диалога:
{history}

Пользователь: {current_message}

text

## Системный промпт
- Хранится в `data.json` (можно редактировать без перезапуска).
- Содержит инструкции: не ставить диагноз, предупреждать о необходимости врача, при экстренных фразах давать телефоны доверия.
- Пример:
Ты — AI-ассистент для первичной психоневрологической диагностики. Твоя задача — задавать уточняющие вопросы, анализировать ответы и давать рекомендации. Никогда не ставь окончательный диагноз. Всегда добавляй дисклеймер: "Это не медицинская консультация. Обратитесь к специалисту." Если пользователь упоминает суицидальные мысли, немедленно сообщи: "Пожалуйста, обратитесь за помощью: телефон доверия 8-800-200-01-22."

text

## Взаимодействие с моделью
- Для диагностики используем `deepseek/deepseek-r1` с инструкцией вернуть JSON.
- JSON должен соответствовать Pydantic модели `DiagnosticReport`.
- При ошибке парсинга логируем и отправляем стандартное сообщение.
